# RTA Junkyard

This directory is for quarantined RTA material from before the refactor reset.

Nothing in here is authoritative. Do not use this directory as the primary
source for refactor requirements, concept definitions, or product direction.
Those live in `/docs/refactor`.

The junkyard exists so old docs, fixtures, prototypes, generated output, and
experiments can be preserved without continuing to define how RTA works.

Use it like this:

- move confusing old material here instead of deleting it immediately,
- copy ideas back out only when they satisfy the new refactor model,
- rewrite promoted material instead of importing stale assumptions blindly,
- keep links from active docs to junkyard files rare and explicit,
- treat every file here as a possible pickup, not source truth.

Promotion out of the junkyard should include:

- a clear reason it belongs in the new model,
- updated naming,
- tests or QA steps,
- documentation in the active refactor packet when the idea is architectural.
